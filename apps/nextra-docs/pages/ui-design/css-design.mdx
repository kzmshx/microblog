## CSS設計の難点

- 規則がちゃんと整理できていない -> 単独での開発時ですら記述の一貫性が損なわれる
- ドキュメント化されていない -> 複数人での開発時に記述の一貫性が損なわれる
- 規則の設計が甘い -> 規則が機能しない

## CSS自体の難点

- グローバルスコープしかない
  - 名前の管理が難しい
- モジュール機能がない
  - スタイルの再利用が難しい
- カスケーディング
  - 影響範囲の制御が難しい

## CSSが難化する理由

そもそも、HTMLは文書を構造化するためのマークアップ言語に過ぎず、CSSは文書を装飾するための言語に過ぎない。

-> 高度化するウェブアプリケーションの要求に追いついていない。

## CSS設計の目的

1. 予測可能性
2. 再利用性
3. 保守性
4. 拡張性

## CSS設計の設計方針

1. 特性に基づいて分類する
2. コンテンツとスタイルを疎結合にする
3. 影響範囲を制限する
4. コンテキストへの依存を制限する
5. 詳細度を制限する
6. クラス名で影響範囲を伝える
7. クラス名で見た目・機能・役割を伝える
8. 拡張性を考慮する
